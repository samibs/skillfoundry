1. **GitHub Orchestrator** (`github-orchestrator.md`)

2. **PR Review Agent** (`pr-review.md`)

   - Analyzes PRs with full diff and file context

   - Zero tolerance for AI-specific issues
   - **Integration**: Directly analyzes GitHub PRs

3. **GitHub Actions Agent** (`github-actions.md`)
   - CI/CD debugging and monitoring

   - **Integration**: Works with git staging area

## 🚀 Workflows

### Example Workflow 1: GitHub Issue → PR

```bash
# 1. Analyze issue
task agent_type="explore" description="Analyze issue" \
  prompt="Use github-orchestrator to analyze issue #123"

# 2. Implement with security
task agent_type="general-purpose" description="Implement" \
  prompt="Use coder to implement with security checks"

# 3. Review changes
task agent_type="code-review" description="Review" \
  prompt="Use pr-review to review staged changes"

# 4. Generate commit message
task agent_type="explore" description="Commit message" \
  prompt="Use commit-message to create conventional commit"
```

### Example Workflow 2: Security Scan & Fix

```bash
# 1. Scan for vulnerabilities
task agent_type="explore" description="Security scan" \
  prompt="Use security-scanner on src/ directory"

# 2. Fix vulnerabilities
task agent_type="general-purpose" description="Fix issues" \
  prompt="Use coder to fix following ANTI_PATTERNS guides"

# 3. Verify fixes
task agent_type="explore" description="Verify" \
  prompt="Use security-scanner to confirm all issues resolved"
```

See `.copilot/WORKFLOW-GUIDE.md` for 5 complete workflows.

## 🔧 Technical Details

### Converter Implementation

**convert-to-copilot.sh**:
- Detects frontmatter in Claude command files
- Converts to Copilot custom agent format
- Preserves agent instructions and capabilities
- Adds usage examples automatically

### Platform Detection

**install.sh**:
- Interactive or flag-based platform selection
- Creates `.framework-platform` marker file
- Copies platform-specific files only
- Installs security documents for both platforms

**update.sh**:
- Reads `.copilot/.framework-platform` to detect platform
- Only updates Copilot-specific files for Copilot projects
- Creates backups in `.claude/backups/YYYYMMDD_HHMMSS/`

## 🎯 Migration Guide

### For Existing Claude Code Projects

**No action required!** Your projects continue working exactly as before.

**Optional**: To get security documents:
```bash
cd ~/your-existing-project
~/dev_tools_20260120_latest/claude_as/update.sh .
```

This adds ANTI_PATTERNS files but doesn't change Claude Code functionality.

### For New Projects

**Choose your platform during installation**:
```bash
cd ~/my-new-project
~/dev_tools_20260120_latest/claude_as/install.sh
# Select: 1) Claude Code  or  2) GitHub Copilot CLI
```

Or use flags:
```bash
~/dev_tools_20260120_latest/claude_as/install.sh --platform=copilot
```
