## What's in This Release

### `/test-map` Skill — Test Documentation Generator

The new `/test-map` skill turns any project's spec files into a client-ready HTML test documentation report. It works in all 5 platforms (Claude Code, Copilot, Cursor, Gemini, Codex).

**How it works:**

1. **Discovery** — Scans for all spec files matching `*.spec.ts`, `*.test.ts`, `*.spec.js`, `*.test.js`, `*_test.py`, `*_spec.rb`, etc.
2. **Classification** — Assigns each test file to one of three value tiers based on what the tests actually verify:
   - **Tier 1 — HIGH VALUE**: Conditional logic, state transitions, business rules, error handling paths, boundary conditions. These tests catch real bugs.
   - **Tier 2 — MEDIUM VALUE**: Dependency injection wiring, HTTP mocking (HttpTestingController), component rendering, integration paths. Infrastructure confidence.
   - **Tier 3 — BASELINE**: Smoke tests (`toBeTruthy()`, `expect(true)`), existence checks, trivial assertions. Low diagnostic value.
3. **Deep analysis** — For Tier 1 and Tier 2 files, reads the test body to extract what each `describe`/`it` block actually verifies. Names ambiguous tests by their actual assertion, not their label.
4. **Assessment** — Computes five quality scores: coverage breadth, test depth, mock discipline, naming clarity, value distribution. Identifies the top improvement opportunities.
5. **HTML output** — Produces a single self-contained HTML file styled to match enterprise test audit standards. Color-coded by tier, exportable, shareable without modification.

**Invoke:** `/test-map` or `/test-map --output report.html` or `/test-map src/app/auth`

### GitHub Copilot + Claude Model Optimization

`.github/copilot-instructions.md` now acts as a full constitutional system prompt for Claude when used via GitHub Copilot Chat. Expanded from ~50 lines to ~160 lines:

- **Identity block** — Establishes the agent as a cold-blooded senior software engineer and agent orchestrator, not a suggestion engine.
- **Constitutional rules** — 12 zero-tolerance rules (no stubs, no TODOs in production code, no `.map()` on nullable arrays, no localStorage tokens, HttpOnly cookies, three-layer completeness enforcement, etc.).
- **Agent invocation table** — Maps every SkillFoundry skill to its `@agentName` invocation pattern for Copilot Chat.
- **Project structure map** — Orients Claude to the SkillFoundry directory layout on first use in any session.
- **Development workflow protocol** — PRD-first, backend-first, implement-test-iterate.
- **Memory storage policy** — Prevents Copilot from writing to its internal `WorkspaceStorage/`; routes all persistent memory to `memory_bank/`.

`.copilot/custom-agents/forge.md` synced to full Claude parity — includes Safeguards 1–6, Phase 2.5, Phase 2.75, and all AC classification logic.

### Fixes

**Config-protect silent skip for `.claude/` entries**

`config-protect.ts` previously threw on allowlisted paths that don't exist on disk (e.g., `.claude/` subdirectories not present in all environments). Now silently skips missing paths rather than surfacing a startup error.

**Version alignment**

All version references corrected after forge orchestrator PR #17 (running from a stale Docker container) reset `sf_cli/package.json`, `mcp-server/src/server.ts`, `mcp-server/src/api/routes.ts`, `README.md`, and both site files from v5.9.0 back to v5.7.1. The corrupted `## [5.7.1]` block injected before the `# Changelog` header in `CHANGELOG.md` has also been removed.

**Root cause:** Forge orchestrator jobs that run in Docker containers must pull the latest main branch before executing. If the container image is stale, version bumps from the host are invisible to the running job, and any PR it creates will contain the old version number. Added to known failure patterns.
