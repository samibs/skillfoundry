Best For

Best for onboarding projects and developers into framework-native execution.

  • Install once into existing repository
  • Create PRD before coding
  • Run go + validation in first session

Anti-Patterns To Avoid

These behaviors reduce framework signal and increase rework risk.

  • Skipping install verification
  • Starting implementation without PRD
  • No test/gate evidence after run

Adoption Plan

  1. Install selected adapters
  2. Author PRD in genesis/
  3. Run implementation command
  4. Validate and document results
