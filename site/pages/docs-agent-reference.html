Best For

Best when task complexity requires multiple agent roles and explicit delegation.

  • Map task to correct primary/secondary agents
  • Use governance agents for approval paths
  • Keep implementation and review roles separate

Anti-Patterns To Avoid

These behaviors reduce framework signal and increase rework risk.

  • Single-agent execution for multi-domain tasks
  • No documented handoff expectations
  • Specialist agents skipped for risky scope

Adoption Plan

  1. Classify objective and risk
  2. Pick orchestrator and executors
  3. Run validation/governance agents
  4. Record decisions and outcomes
